My idea is to create a service (or the approppriate word) that will be composed of 2 parts, a telegram bot and a file converter.
The bot will be waiting for a user to send them a file, if the file is of type epub/mobi/azw3, convert it to one that an amazon kindle can take, and then, if an amazon kindle is connected to the server, transfer it to the approppriate folder. It'd be best if this could be dockerized, so that redeploying when updating any of the components is not a pain in the ass. During the conversion process, the bot should send a message stating that the conversion is taking place, and once finished another message should be sent. When the file has finished transfer to the kindle, send a message stating that the user can now retire the device
